# [cite_start]Java OOP Final Project Requirements [cite: 1]

[cite_start]You are required to design and implement a Java Swing-based GUI application that demonstrates a real-world system using core Object-Oriented Programming (OOP) principles[cite: 2].

## [cite_start]Core Project Requirements [cite: 3]

[cite_start]Your project must include the following[cite: 4]:

### [cite_start]1. Graphical User Interface (GUI) [cite: 5]
* [cite_start]Implemented using Java Swing[cite: 6].
* [cite_start]The interface must be menu-driven with clearly labeled buttons, menus, or tabs[cite: 7].
* [cite_start]Include forms for input (e.g., adding students or courses) and display areas for viewing stored data[cite: 8].

### [cite_start]2. Object-Oriented Design [cite: 9]
[cite_start]Your program must demonstrate all four core OOP principles[cite: 10]:

* [cite_start]**Encapsulation**[cite: 11]:
    * [cite_start]Use private fields and provide public getter/setter methods[cite: 12].
    * [cite_start]Avoid exposing internal details directly[cite: 13].
* [cite_start]**Inheritance**[cite: 14]:
    * [cite_start]Create subclasses that extend a superclass[cite: 15].
* [cite_start]**Polymorphism**[cite: 16]:
    * [cite_start]Demonstrate method overriding[cite: 17].
    * [cite_start]Use dynamic method dispatch[cite: 18].
    * [cite_start]May include interface-based or abstract class-based polymorphism[cite: 19].
* [cite_start]**Abstraction**[cite: 20]:
    * [cite_start]Use abstract classes and/or interfaces[cite: 21].

### [cite_start]3. Exception Handling [cite: 22]
* [cite_start]Use try-catch blocks to handle possible runtime errors[cite: 23].
* [cite_start]Implement at least one custom exception[cite: 24].

### [cite_start]4. File Handling [cite: 25]
* [cite_start]Your application must be able to save and load data to/from a file[cite: 27].
* [cite_start]Use `FileReader`, `BufferedReader`, `FileWriter`, or `PrintWriter`[cite: 28].

---

## [cite_start]Minimum Deliverables [cite: 29]

1.  [cite_start]Java source code with clear comments[cite: 30].
2.  [cite_start]GUI application (with working menus and forms)[cite: 31].
3.  [cite_start]Sample input/output files used for file handling[cite: 32].
4.  [cite_start]A project report explaining how your code fulfills all required OOP principles and group members' contributions[cite: 33].

## [cite_start]Important Notes [cite: 34]

* [cite_start]All classes must follow proper OOP design[cite: 35].
* [cite_start]GUI must be fully functional and user-friendly[cite: 36].
* [cite_start]Code duplication should be avoided[cite: 37].
* [cite_start]Projects not demonstrating all OOP principles will be penalized[cite: 38].
